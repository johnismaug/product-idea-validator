# product-idea-validator · Skill README

> 专为独立开发者 / 小团队设计的产品点子验证与打磨 Skill

## 这个 Skill 是做什么的

`product-idea-validator` 帮助独立开发者和小团队在动手开发之前，用结构化的方式想清楚三个核心问题：

- **值不值得做？** 需求是否真实存在，市场规模是否够用
- **卖给谁？** 精准定位目标用户，描述他们的痛点与支付意愿
- **怎么卖？** 差异化策略、定价区间、MVP 范围

---

## 触发方式

以下任意表达都会自动触发本 Skill：

| 触发词示例 | 场景 |
|---|---|
| 我想做一个… | 有新 idea，想知道值不值得做 |
| 帮我分析这个产品 | 已有方向，需要结构化分析 |
| 这个需求值不值得做 | 对需求有疑虑，需要验证 |
| 帮我想想能卖给谁 | 用户定位模糊 |
| 帮我打磨产品方向 | 方向粗糙，需要打磨细化 |
| 有人买单吗 | 商业可行性疑问 |
| MVP 怎么做 / 怎么定价 | 执行层问题 |

> 即使用户只是随口说「我想做个记单词的小程序」，也会触发本 Skill。

---

## 输出结构

```
1. 问题与需求定义
2. 目标用户画像
3. 市场规模估算（TAM/SAM/SOM）
4. 差异化切入点
5. MVP 定义与优先级
6. 商业模式与定价区间
7. 验证行动清单（低成本 pre-dev 动作）
```

## 适用场景

✅ 工具类小程序 / APK / SaaS 微工具，独立开发者 side project 立项前评估
❌ 大型平台级产品战略，纯技术可行性评估
